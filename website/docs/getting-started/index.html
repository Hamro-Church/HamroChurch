Getting Started

Install, launch, and prepare Hamro Church for real services.

This guide covers the first-run workflow your projection or media team should complete before using Hamro Church in a live service.

System requirements

Hamro Church works best on a modern Windows, macOS, or Linux computer with reliable display output support, enough RAM for media playback, and a fast local drive for project content.

  • Use a dedicated presentation machine if possible.
  • Prefer local SSD storage for media and Bible libraries.
  • Verify GPU, display adapters, and audio routing before service day.
Recommendation: Keep your projection computer focused on presentation work. Extra background applications often cause playback or output issues.

Installation

Download the correct build for your operating system from the Hamro Church download page or the GitHub releases page. Run the installer, then launch the app once to let the initial setup complete.

  1. Open Download and choose the Windows, macOS, or Linux build.
  2. Install using administrator privileges if your environment requires it.
  3. Restart the computer if the operating system requests it after install.

First launch

On first launch, choose whether the team wants to begin in English or Nepali. After that, confirm basic settings before creating your first live project.

  • Pick the language that matches your operators.
  • Check the default theme and font behavior.
  • Open the About or Update panel to confirm the version you installed.

If you are migrating from another local setup, import or copy your required Bible, hymn, and media assets before building a service from scratch.

Content location

Decide where your church will store ongoing media, hymns, custom Bible files, and service projects. Consistency matters more than clever folder structures.

  • Keep the main content location on a stable local drive.
  • Avoid storing active content only on removable media.
  • If multiple operators work on different machines, decide how files are shared and verified.

First service checklist

Before using Hamro Church in a real service, run a short rehearsal that verifies the full flow from the operator screen to the audience output.

  1. Create a small sample project with a song, a scripture reading, and a media item.
  2. Test the main audience output and confirm the correct screen is assigned.
  3. Open stage view or remote control if your church uses assistant operators.
  4. Confirm Nepali text, digits, and scripture content display correctly where needed.
  5. Make one manual backup of the content folder before building the full service library.
